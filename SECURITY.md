# Security Policy

## Reporting a Vulnerability

If you discover a security vulnerability in this project, please report it responsibly.

**Please do NOT open a public GitHub issue for security vulnerabilities.**

Instead, send an email to the repository maintainer with:

- A description of the vulnerability
- Steps to reproduce
- Potential impact

## Supported Versions

| Version | Supported          |
| ------- | ------------------ |
| latest  | :white_check_mark: |

## Security Best Practices

This project follows these security practices:

- No credentials or secrets are stored in the repository
- `.gitignore` excludes sensitive user-specific files
- Build artifacts are excluded from version control
